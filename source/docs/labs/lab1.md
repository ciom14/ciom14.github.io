# Лабораторная работа 1

**Тема:** Создание и развёртывание статического сайта на базе MkDocs 
с публикацией на GitHub Pages

---

## 1. Цель работы

- Освоить процесс создания статического сайта с использованием генератора документации MkDocs.
- Научиться организовывать структуру документации проекта (портфолио лабораторных работ).
- Изучить базовые принципы работы с системой контроля версий Git и платформой GitHub.
- Развернуть статический сайт с использованием механизма GitHub Pages на домене вида username.github.io.
- Освоить базовую настройку темы оформления и конфигурационного файла mkdocs.yml.
---

## 2. Задание

Создать публичный репозиторий на GitHub, настроить MkDocs с темой оформления,
создать не менее 5 страниц и опубликовать сайт через GitHub Pages из каталога `/docs`.

---

## 3. Выбор темы оформления

Для оформления сайта выбрана тема **Material for MkDocs**.

Обоснование выбора:

- **Тёмная цветовая схема (slate)** — снижает нагрузку на глаза при длительной работе
- **Профессиональный внешний вид** — тема используется в документации крупных 
  реальных проектов: FastAPI, SQLModel
- **Поддержка русского языка** — интерфейс и поиск корректно работают с кириллицей
- **Удобная навигация** — вкладки сверху и боковое меню делают структуру сайта 
  понятной и удобной для пользователя
- **Переключение тем** — встроенная кнопка позволяет переключаться между 
  тёмным и светлым режимом
- **Не требует дополнительной настройки** — тема хорошо выглядит сразу 
  после подключения
